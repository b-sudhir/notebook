HTML

  • HTML (HyperText Markup Language) is a markup language that tells web browsers how to structure the web pages you visit. It can be as complicated or as simple as the web developer wants it to be. HTML consists of a series of elements, which you use to enclose, wrap, or mark up different parts of content to make it appear or act in a certain way. The enclosing tags can make content into a hyperlink to connect to another page, italicize words, and so on.
  • No matter what browser you will be using, Chrome, Safari, Edge, Brave, or Firefox, all these applications do the same thing. They take different files such as HTML, CSS, and JavaScript and render them into a website. But even though most modern websites are created using these three different types combined, we can’t create a website with just a CSS file or JavaScript file. But we can create a website using an HTML file
  • HTML defines the content and structure of the website
  • HTML stands for HyperText Markup Language
    • HyperText refers to the pieces of text which can link to other documents in the website. So, these pieces of text are hypertext or hyperlinks and they are the foundation of how an HTML website works. If we click on a hyperlink on a website and it will take us to another HTML page or website and that’s how hyperlinks work
    • Markup Language refers to the manuscript used to mark and show different things, for example, which parts need to be bold, italics, etc. Markup in HTML is done through the HTML tags
Anatomy of HTML Tags
  • <h1>Hello World</h1> - <h1>is the start tag and </h1> is the end tag. In between the opening tag and the closing tag is the content that we want to render
  • In HTML, we have some tags without the end tag, and those HTML tags are called Self Closing Tags, E.g., <br>, <hr>, etc.
  • In HTML, some tags will have parameters (aka. attributes), for example: <hr size=”3”>, where <hr> is the HTML self-closing tag and size=”3” is the attribute of that HTML tag and it gives more information to the browser to specify modifications to that HTML element

    • Note: To know whether the HTML element is a self-closing tag or the HTML element is having the end tag and also if you want to know the attributes of an HTML element, then kindly refer to the documentation (reference: devdocs.io)

HTML Boilerplate

An HTML Boilerplate includes the basic structure for an HTML document, like typical HTML, head, and body tags and as well as links to CSS and JavaScript files.

                    
                        <!DOCTYPE html>
                        <html lang="en">
                        <head>
                            <meta charset="UTF-8">
                            <meta http-equiv="X-UA-Compatible" content="IE=edge">
                            <meta name="viewport" content="width=device-width, initial-scale=1.0">
                            <title>Document</title>
                        </head>
                        <body>
                            
                        </body>
                        </html>

The above HTML code is the boilerplate for HTML and with that we usually start off with any new HTML file

  • <!DOCTYPE html> - is the doctype declaration of HTML5 and this tag tells the browser which version of HTML the file was written in
  • <html lang="en"></html> - is the root element of the document i.e., all the HTML element tags will go inside this <html> tag and this tag has an attribute called lang="en", where it says the language of the text content in that element
  • <head></head> - is the tag containing important information about our website and this important information will not be displayed to the user
  • <meta charset="UTF-8"> - is a self-closing tag for the character set encoding of the webpage. This tag ensures that the character that you’re using on your website is getting displayed correctly and UTF-8 is the charset we are using
  • <title></title> - this tag is responsible for displaying the title of our website on the tab bar
  • <body></body> - in between this tag is where all the content of the website goes (text, titles, images, etc.) and whatever you write in between this tag, those content/information will be displayed on the website
HTML Tags
Main Root & Sectioning Root Tags
  • <!DOCTYPE html> - declaration is not an HTML tag. It is an "information" to the browser about what document type to expect.
  • <!DOCTYPE html> - is the doctype declaration of HTML5 and this tag tells the browser which version of HTML the file was written in
  • In older documents (HTML 4 or XHTML), the declaration is more complicated because the declaration must refer to a DTD (Document Type Definition).
    • HTML 4.01 - <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
    • XHTML 1.1 - <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

  • The <head> element is a container for metadata (data about data) and is placed between the <html> tag and the <body> tag
  • Metadata is data about the HTML document. Metadata is not displayed.
  • Metadata typically define the document title, character set, styles, scripts, and other meta information.
  • The following elements can go inside <head> element:
    • <title>
    • <style>
    • <base>
    • <link>
    • <meta>
    • <script>
    • <noscript>

  • The <title> tag defines the title of the document. The title must be text-only, and it is shown in the browser's title bar or in the page's tab.
  • The <title> tag is required in HTML documents.
  • The contents of a page title is very important for search engine optimization (SEO). The page title is used by search engine algorithms to decide the order when listing pages in search results.
  • The <title> element:
    • Defines a title in the browser toolbar
    • Provides a title for the page when it is added to favorites
    • Displays a title for the page in search-engine results
  • Here are some tips for creating a title:
    • Go for a longer, descriptive title (avoid one- or two-word titles)
    • Search engines will display about 50-60 characters of the title, so try not to have titles longer than that
    • Do not use just a list of words as the title (this may reduce the page's position in search results)
